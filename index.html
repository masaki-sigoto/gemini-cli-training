Gemini CLI 実務活用講座

ターミナル / PowerShell からAIを起動し、実務を自動化する3時間講座

この講座について

本講座は、AI（Gemini CLI）を実務で活かすための実践的なハンズオン研修です。ターミナルやPowerShellからAIに指示を出し、日々の業務で発生するファイル整理・資料作成・データ集計を自動化する一連の流れを体験します。

⏱️ セットアップ時間について

初めて受講される方は、Node.js や Gemini CLI のインストールに 10分〜1時間程度かかる場合があります。会社PCをお使いの方は特に注意が必要です。前日までに事前準備を完了されることを強くお勧めします。

受講にあたって知っておいてほしいこと

  • 本講座はAI活用の実践的な流れをつかむことを目的としています。コマンドをすべて暗記する必要はありません
  • 演習ではすべて練習用サンプルデータを使用しますので、実務データが外部に漏れる心配はありません
  • 会社支給PCをご利用の方は、インストール制限がある場合があります。情報システム部門への事前確認をお願いします
  • エラーが出ても慌てないでください。講師・サポートスタッフがリアルタイムで対応します
  • 質問はいつでも歓迎です。チャットへの書き込みや音声での質問が可能です

⏱️ セットアップには時間がかかります

Gemini CLI を使えるようにするまでの準備は、環境によって 10分〜1時間程度 かかります。当日ぎりぎりではなく、前日までに完了しておくことを強くお勧めします。特に会社PCをご利用の方は、早めにご確認ください。

事前に済ませておいてほしいこと（チェックリスト）

受講前日までに、以下の項目をすべて確認してください。不安な点があれば事前にご連絡ください。

PC・環境

  • 使用するPCを決めておく
    自宅PC・個人PC・会社PCのどれを使うか決めてください。会社PCの場合は次の項目も必ず確認してください。
  • 会社PCの場合: ソフトウェアのインストール制限を確認する
    情報システム部門（IT部門）に「Node.js と npm をインストールしてよいか」を事前に確認してください。制限がある場合は個人PCでの参加をご検討ください。
  • PCの管理者権限（Administratorアカウント）を確認する
    ソフトウェアのインストールに管理者権限が必要です。Windowsの場合は「設定 → アカウント」で確認できます。Macの場合は「システム設定 → ユーザとグループ」で確認してください。
  • 安定したインターネット回線に接続できる状態にする
    ダウンロードや AI との通信が発生します。できれば有線LAN、または安定したWi-Fiを確保してください。
  • 充電器を用意する（またはバッテリー残量を確認）
    3時間の講座中、充電が切れると復旧に時間がかかります。充電器を接続した状態でご参加ください。

Googleアカウント

  • Googleアカウントを持っていることを確認する
    Gemini CLI のログインにはGoogleアカウントが必要です。お持ちでない場合は accounts.google.com で無料作成できます。
  • 使用するPCのブラウザでGoogleアカウントにログインしておく
    Chrome、Firefox、Edge などで google.com にアクセスし、右上のアイコンでログイン済みであることを確認してください。
  • 二段階認証の設定を確認する
    二段階認証を設定している場合、CLIログイン時にスマートフォンでの承認が必要です。スマートフォンを手元に用意しておいてください。

ターミナル / PowerShell

  • ターミナル（Mac）またはPowerShell（Windows）が開けることを確認する
    Macは「Spotlight検索（Command + Space）」で「ターミナル」と入力して起動できます。Windowsはスタートメニューで「PowerShell」と検索して起動できます。
  • Node.js と npm がインストールできる状態であることを確認する
    可能であれば、セットアップページの手順に従って事前にインストールを完了しておいてください。時間短縮になります。
  • Windowsの場合: PowerShellのスクリプト実行ポリシーを確認する
    PowerShellで Get-ExecutionPolicy を実行し、「Restricted」と表示された場合はトラブルシューティングガイドを参照してください。

セキュリティ・その他のソフトウェア

  • セキュリティソフトの設定を確認する
    ウイルス対策ソフトがnpmのインストールをブロックする場合があります。会社のセキュリティポリシーの範囲内で、一時的に設定を調整できるか確認しておいてください。
  • Word・Excel・PowerPointが開ける状態であることを確認する
    Microsoft Officeがインストールされていることを確認してください。ない場合は無料のLibreOfficeまたはGoogle Docs/Sheetsで代替できます。
  • ファイルを保存する場所（フォルダ）を決めておく
    デスクトップ上に「gemini-practice」フォルダなどを作っておくと、演習中のファイル管理がスムーズです。

Zoom / Google Meet（受講環境）

  • Zoom または Google Meet が正常に動作することを確認する
    カメラ・マイク・スピーカーが使える状態にしておいてください。接続テストを事前に行うことをお勧めします。
  • PCの再起動ができる状態にしておく
    インストール後に再起動が必要になる場合があります。保存中の作業ファイルは事前に閉じておくと安心です。

受講者へのお願い

1 開始10〜15分前には入室をお願いします
接続確認・マイクテスト・画面共有の確認など、開始前のウォームアップに時間を使います。開始ぎりぎりに入室すると、セットアップが遅れる場合があります。
2 事前準備は前日までに完了してください
Node.js・Gemini CLIのインストールは当日に行うと時間がかかり、他の受講者の進行に影響が出る場合があります。セットアップページを参考に、前日までに準備を完了してください。
3 会社PCをお使いの方は、情報システム部門へ事前確認をお願いします
Node.js・npm・Gemini CLIのインストール許可、PowerShellのスクリプト実行ポリシー変更が必要な場合があります。「業務効率化のためのAIツール研修に使用する」旨を伝えて確認してください。
4 エラーが出ても画面を閉じずに、そのまま共有してください
エラーメッセージは問題解決のための重要な情報です。慌てて閉じたり、再起動したりせず、エラー画面のスクリーンショットを撮るか、チャットにエラー内容を貼り付けてください。
5 演習フォルダ以外のファイルは操作しないようにしてください
講座では専用の演習フォルダ（sample_data_messy など）を使います。それ以外の場所のファイルを誤って削除・移動しないよう注意してください。
6 実務データ（個人情報・社内データ）は使用しないでください
演習には必ずサンプルデータを使用してください。AIへの入力に実在する顧客情報・社内機密情報を含めることは、セキュリティ上のリスクになります。
7 わからないことはその場で質問してください
チャットへの書き込みや音声での質問を歓迎します。「こんなこと聞いていいのか」という遠慮は不要です。同じ疑問を持っている方が必ずいます。
8 デスクトップを整理しておくことをお勧めします
画面共有をする場合、デスクトップが見えます。個人情報や機密ファイルのアイコン・ファイルが表示されないよう、事前に整理・移動しておくことをお勧めします。

macOS 向け

Mac セットアップガイド

Gemini CLI を Mac で使えるようにするための手順を、一つひとつ丁寧に説明します。

1

ターミナルを開こう

「ターミナル」とは、Mac に文字を打ち込んで命令を与えるアプリです。 普段使うマウス操作の代わりに、キーボードでコンピューターと会話する窓口だと思ってください。

開き方 ① Spotlight 検索（いちばん簡単）

  1. キーボードの Command (⌘) + Space を同時に押します
  2. 画面中央に検索バーが現れます
  3. 「ターミナル」 と入力します
  4. 「ターミナル.app」が表示されたら Enter を押します

開き方 ② Launchpad から開く

  1. Dock（画面下のバー）にある ロケットのアイコン（Launchpad） をクリック
  2. 「その他」 フォルダを開く
  3. 「ターミナル」 をクリック

開き方 ③ Finder から開く

  1. Finder を開く（画面左下の笑顔のアイコン）
  2. 左サイドバーの 「アプリケーション」 をクリック
  3. 「ユーティリティ」 フォルダを開く
  4. 「ターミナル」 をダブルクリック
ターミナルが開いたら？
黒や白の画面に username@MacBook ~ % のような文字が表示されれば成功です。 この % または $ マークの後ろにコマンドを入力します。
2

基本コマンドを体験しよう

まずはターミナルの基本操作を練習しましょう。 コマンドを入力したら Enter キーを押して実行します。

現在地を確認する（pwd）

「今、自分はコンピューターのどこにいるのか」を確認するコマンドです。

$ pwd

実行すると /Users/あなたの名前 のようなパスが表示されます。 これが「現在地（ホームディレクトリ）」です。

ファイルやフォルダの一覧を見る（ls）

現在地にあるファイルやフォルダの一覧を表示します。Finder で中身を見るのと同じです。

$ ls

Desktop、Documents、Downloads などのフォルダ名が表示されれば OK です。

デスクトップに移動する（cd）

cd は「Change Directory（場所を変える）」の略です。

$ cd ~/Desktop

~/ は「ホームディレクトリ」を意味します。 コマンド実行後、pwd を打つと /Users/あなたの名前/Desktop になっているはずです。

作業フォルダを作る（mkdir）

講座で使う専用フォルダをデスクトップに作りましょう。

$ mkdir gemini-workshop

Finder でデスクトップを見ると「gemini-workshop」フォルダが作成されているはずです。

作業フォルダに移動する

$ cd gemini-workshop

pwd を打って /Users/あなたの名前/Desktop/gemini-workshop と表示されれば成功です。

注意：ダウンロードフォルダと混同しないように
作業ファイルはデスクトップの gemini-workshop フォルダに保存しましょう。 Downloads フォルダに保存してしまうと、後で迷子になりやすいです。
3

Node.js を確認・インストールしよう

Gemini CLI は「Node.js」という仕組みの上で動きます。 まずインストール済みかどうか確認しましょう。

Node.js が入っているか確認

$ node --version

v20.11.0 のようなバージョン番号が表示されれば、すでにインストール済みです。 次の手順（インストール）はスキップして Step 4 へ進んでください。

Node.js が入っていない場合のインストール方法

「command not found」と表示された場合はインストールが必要です。

  1. ブラウザで https://nodejs.org を開く
  2. 「LTS（推奨版）」 と書かれたボタンをクリックしてダウンロード
  3. ダウンロードされた .pkg ファイルをダブルクリック
  4. インストーラーの指示に従って「続ける」「インストール」をクリック
  5. インストール完了後、ターミナルを一度閉じて再度開く
  6. 再度 node --version を実行してバージョンが表示されることを確認

npm（パッケージマネージャー）も確認

npm は Node.js と一緒にインストールされるツールです。

$ npm --version

10.2.4 のようなバージョン番号が表示されれば OK です。

4

Gemini CLI をインストールしよう

いよいよ Gemini CLI をインストールします。 npm というコマンドを使って、インターネットから自動的にダウンロード・インストールされます。

インストールコマンドを実行

$ npm install -g @google/gemini-cli

-g は「グローバル（Global）」の意味で、Mac 全体で使えるようにインストールします。 しばらく待つと完了メッセージが表示されます。

インストール中にエラーが出た場合
sudo npm install -g @google/gemini-cli のように先頭に sudo を付けて実行してください。 パスワードを求められたら Mac のログインパスワードを入力します（入力中は文字が表示されませんが正常です）。

インストールできたか確認

$ gemini --version

バージョン番号が表示されればインストール成功です。

5

Gemini CLI を起動しよう

起動コマンドを実行

$ gemini

初回起動時：Google アカウントでログイン

初めて起動するときは、Google アカウントでの認証が必要です。

  1. ターミナルに認証用の URL またはブラウザ起動の案内が表示されます
  2. ブラウザが自動で開く（または表示された URL をブラウザで開く）
  3. 使用する Google アカウント を選択してログイン

Windows セットアップガイド

Gemini CLI を Windows で使えるようにするための手順を、一つひとつ丁寧に説明します。

1

PowerShell を開こう

「PowerShell」とは、Windows に文字を打ち込んで命令を与えるアプリです。 普段使うマウス操作の代わりに、キーボードでコンピューターと会話する窓口だと思ってください。

開き方 ① スタートメニューから検索（いちばん簡単）

  1. 画面左下の スタートメニュー（Windowsアイコン） をクリック
  2. 検索ボックスに 「PowerShell」 と入力
  3. 「Windows PowerShell」または「PowerShell」が表示されたらクリック

開き方 ② ショートカットキーから開く

  1. Windows キー + X を同時に押す
  2. 表示されたメニューから 「Windows PowerShell」 をクリック
コマンドプロンプト（cmd）との違い
Windows には「コマンドプロンプト（cmd.exe）」という古い画面もありますが、 この講座では PowerShell を使います。 PowerShell の方が高機能で、現代的なコマンドが使えます。 PowerShell は青い背景が目印です（環境によって異なる場合があります）。
PowerShell が開いたら？
PS C:\Users\あなたの名前> のような文字が表示されれば成功です。 この > マークの後ろにコマンドを入力します。
2

基本コマンドを体験しよう

まずはターミナルの基本操作を練習しましょう。 コマンドを入力したら Enter キーを押して実行します。

現在地を確認する（pwd）

「今、自分はコンピューターのどこにいるのか」を確認するコマンドです。

PS> pwd

実行すると C:\Users\あなたの名前 のようなパスが表示されます。 これが「現在地（ホームディレクトリ）」です。
※ Get-Location と入力しても同じ結果が得られます。

ファイルやフォルダの一覧を見る（ls / dir）

現在地にあるファイルやフォルダの一覧を表示します。エクスプローラーで中身を見るのと同じです。

PS> ls

Desktop、Documents、Downloads などのフォルダ名が表示されれば OK です。
※ dir と入力しても同じ結果が得られます。

デスクトップに移動する（cd）

cd は「Change Directory（場所を変える）」の略です。

PS> cd ~\Desktop

~\ は「ホームディレクトリ」を意味します。 コマンド実行後、pwd を打つと C:\Users\あなたの名前\Desktop になっているはずです。

作業フォルダを作る（mkdir）

講座で使う専用フォルダをデスクトップに作りましょう。

PS> mkdir gemini-workshop

エクスプローラーでデスクトップを見ると「gemini-workshop」フォルダが作成されているはずです。

作業フォルダに移動する

PS> cd gemini-workshop

pwd を打って C:\Users\あなたの名前\Desktop\gemini-workshop と表示されれば成功です。

注意：OneDrive とローカルフォルダの混在
Windows では「デスクトップ」が OneDrive と同期されている場合があります。 作業フォルダは C:\Users\あなたの名前\Desktop\（ローカル）に作ることを推奨します。 OneDrive 配下（C:\Users\あなたの名前\OneDrive\Desktop\）に作ると 同期の問題が起きることがあります。

Node.js が入っていない場合のインストール方法

  3. ダウンロードされた .msi ファイルをダブルクリック
  4. インストーラーの「Next」「Install」をクリックして進める
  5. 「Install」ボタンを押すと管理者権限の確認が出る場合があります →「はい」をクリック
  6. インストール完了後、PowerShell を一度閉じて再度開く
  7. 再度 node --version を実行してバージョンが表示されることを確認

npm（パッケージマネージャー）も確認

PS> npm --version

10.2.4 のようなバージョン番号が表示されれば OK です。

Windows の PowerShell には「実行ポリシー」という安全設定があります。 スクリプトが実行できるよう、設定を確認・変更します。

症状

解決方法 ①：Spotlight 検索を使う（最も簡単）

  1. キーボードの Command (⌘) + Space を同時に押す

  1. Dock のロケットアイコン（Launchpad）をクリック

2. 「command not found: node」が表示される

症状
node --version を実行したときに zsh: command not found: node と表示される。

原因

Node.js がインストールされていないか、インストールが完了していない状態です。

解決方法

  1. ブラウザで https://nodejs.org を開く
  2. 「LTS（推奨版）」 ボタンをクリックしてダウンロード
  3. ダウンロードされた .pkg ファイルをダブルクリック
  4. インストーラーに従って「続ける」「インストール」をクリック
  5. ターミナルを一度完全に閉じる（Cmd+Q）
  6. ターミナルを再度開く
  7. node --version を実行してバージョンが表示されることを確認
ポイント
インストール後はターミナルを完全に終了してから再度開くことが重要です。 ターミナルを閉じただけ（Cmd+W）では不十分な場合があります。 Dock のターミナルアイコンを右クリック→「終了」で完全に終了できます。

3. 「command not found: gemini」が表示される

症状
gemini と入力したときに zsh: command not found: gemini と表示される。

原因と解決方法

原因 A：npm install がまだ実行されていない

以下のコマンドでインストールを実行してください。

$ npm install -g @google/gemini-cli

原因 B：インストール後にターミナルを再起動していない

インストール完了後にターミナルを再起動（Cmd+Q で完全終了 → 再度起動）してから 再度 gemini --version を試してください。

原因 C：PATH が通っていない

npm のグローバルインストール先がパスに含まれていない可能性があります。

$ npm bin -g

上のコマンドで表示されたパス（例: /usr/local/bin）が PATH に含まれているか確認します。

$ echo $PATH

npm のパスが表示されない場合は、~/.zshrc に追記が必要です。 詳しくは講師に確認してください。

原因 D：インストールにエラーが出ていた

sudo を付けて再インストールを試みてください。

$ sudo npm install -g @google/gemini-cli

現在地を Finder で開きます。 Finder のウィンドウが開き、どのフォルダにいるか視覚的に確認できます。

解決方法 ③：ホームディレクトリに戻る

迷ったときはホームディレクトリ（出発点）に戻りましょう。

$ cd ~

その後、改めて cd ~/Desktop/gemini-workshop で作業フォルダに移動してください。

7. Downloads フォルダに保存してしまった

$ mv ~/Downloads/フォルダ名 ~/Desktop/gemini-workshop/
Finder でも移動できます

  2. Finder のウィンドウ下部に現在のパスが表示される（表示されない場合は「表示」→「パスバーを表示」をクリック）

PS> npm install -g @google/gemini-cli

4. 実行ポリシーエラーが表示される

症状
スクリプトを実行しようとすると以下のようなエラーが表示される：
このシステムではスクリプトの実行が無効になっているため、ファイル ... を読み込むことができません。

原因

現在の実行ポリシーを確認

「実行ポリシーを変更しますか？」と聞かれたら Y を入力して Enter を押します。

会社 PC では管理者に確認してから実行してください

6. OneDrive 配下で問題が起きる

症状
作業フォルダを OneDrive と同期されているデスクトップに作成したため、 ファイルのロックや同期の競合が起きる。

原因

Windows の「デスクトップ」フォルダが OneDrive と同期されている場合、 OneDrive がファイルをロックすることがあります。 この状態で CLI ツールがファイルを操作しようとすると競合が起きます。

解決方法：作業フォルダをローカルに移動する

  1. ローカルの作業フォルダを作成する
PS> mkdir C:\Users\$env:USERNAME\gemini-workshop
  1. 作業フォルダに移動する
PS> cd C:\Users\$env:USERNAME\gemini-workshop

これで OneDrive の影響を受けない場所に作業フォルダが作成されます。

OneDrive かどうかを確認する方法

PS> pwd

パスに OneDrive が含まれている場合（例: C:\Users\名前\OneDrive\Desktop）は OneDrive 配下にいます。上記の手順でローカルに移動してください。

8. 文字化けする

症状
PowerShell で日本語が正しく表示されず、文字化けしている。 例：「縺薙s縺ォ縺。縺ッ」のような表示になる。

原因

PowerShell の文字コードが UTF-8 ではなく Shift-JIS などになっています。

解決方法：文字コードを UTF-8 に変更する

PS> chcp 65001

65001 は UTF-8 のコードページ番号です。 実行後、文字化けが解消されるか確認してください。

永続的に設定したい場合

PowerShell のプロファイルに設定を追加することで、毎回自動的に UTF-8 になります。

PS> Add-Content $PROFILE "`n[Console]::OutputEncoding = [Text.Encoding]::UTF8"

この設定は次回 PowerShell を起動したときから有効になります。

1. Gemini CLI とは何か

Gemini CLI（ジェミナイ・シーエルアイ）は、Google が提供する AI アシスタント「Gemini」を、あなたの PC 上のターミナル（黒い画面）や PowerShell から直接呼び出せるツールです。

Gemini CLI でできること

  • ファイル・フォルダの整理：散らかったファイルを分類・リネーム・移動

失敗例2：確認なしで実行させる
「すぐにファイルを移動して」と一気に指示すると、 元のファイルが失われたり意図しない場所に移動することがあります。
対策：「実行前に何をするか一覧で見せてください」を必ず入れましょう。
失敗例3：実務データをそのまま使う
研修の学習段階で本番の顧客データや機密情報をGemini CLIに入力するのは危険です。
対策：練習・検証はサンプルデータで行い、本番導入は会社のポリシーを確認してから。
失敗例4：一度に大量の指示を出す
「Excelを作って、分析して、Wordに書いて、スライドも作って」と一気に依頼すると、 どこかでエラーが起きても原因が特定しにくくなります。
対策：1つずつ順番に依頼し、それぞれ確認してから次に進みましょう。

4. 安全に使う4つのポイント

1

必ず確認してから実行する

ファイルの移動・削除・上書きを伴う操作は、実行前に「何をするか一覧で見せてください」と確認を取る癖をつけましょう。 Gemini CLI は確認なしで実行しようとする場合もありますが、「先に計画を教えてから実行してください」と一言添えるだけで安全性が大幅に上がります。

2

バックアップを取ってから作業する

大切なデータは Gemini CLI に触れさせる前に必ずバックアップを作成してください。 「元データは残してコピー側で整理してください」というプロンプトを習慣化することも有効です。

3

最初は練習データで試す

新しい操作を試すときは、本番データではなくサンプルデータで動作確認してから本番に適用してください。 今日の研修で使った sample_data_messy のような練習用フォルダを常に用意しておくと安心です。

4

小さく始めて少しずつ拡大する

最初から全業務をAI化しようとせず、「まず1つの定型業務をAIで試してみる」ところから始めましょう。 うまくいったら少しずつ対象を広げていく方が、失敗リスクが低く学習効果も高まります。

5. AI利用の基本的な注意点

Gemini CLI を含むすべてのAIツールを利用する際の基本的な注意事項です（OWASP AIリスク軽量版に基づく）。

個人情報・機密情報を入力しない
顧客の氏名・住所・電話番号、社内の未公開財務データ、契約情報などはAIに入力しないでください。 入力したデータがモデルの学習に使われる可能性があります。 社内ポリシーと利用規約を必ず確認してください。
AIの出力を鵜呑みにしない
AIは「もっともらしい文章」を生成しますが、事実と異なる情報（ハルシネーション）を出力することがあります。 数値・法律・医療・専門知識に関する出力は、必ず人間が内容を確認・検証してから使用してください。
重要な意思決定はAIだけに頼らない
今日の演習で行った予測・戦略提案はあくまでも参考情報です。 採用・予算配分・法的判断など重要な意思決定は、必ず専門家・上長の判断と組み合わせてください。
プロンプト・出力内容の管理
AIとのやり取り（プロンプト・レスポンス）も業務情報の一部です。 スクリーンショットや共有には社内のセキュリティポリシーに従ってください。
